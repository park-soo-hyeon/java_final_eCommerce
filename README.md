# 🛒 파일 기반 e-Commerce 시스템 메뉴 구조 및 실행 설명서

<br>

## 🤖 1. LLM (AI) 활용 참조

본 과제의 Java 코드를 개발하는 과정에서 아래와 같은 특정 자바 문법 및 시스템 구조(아키텍처)를 학습하고 해결하기 위해 **제미나이(Gemini) AI**를 참조하여 구현하였습니다.

- **스캐너 버퍼 버그 해결:** `scanner.nextInt()` 사용 시 발생하는 엔터(`\n`) 문제를 학습하고, `Integer.parseInt(scanner.nextLine())` 문법으로 통일하여 입력 안정성을 높이는 방법을 참조했습니다.
- **데이터 수정 시 빈 값(Enter) 처리:** 데이터를 수정할 때 입력값 없이 엔터만 쳤을 경우(`isEmpty()`), **삼항 연산자**를 활용하여 기존 데이터를 유지하는 문법 패턴을 학습하여 적용했습니다.
- **파일 DB 메모리 동기화:** 장바구니에 담은 데이터가 주문 컨트롤러에서 읽히지 않는 문제를 해결하기 위해, Repository 클래스의 List 변수에 `static` 키워드를 적용하여 프로그램 전역에서 단일 메모리를 공유하도록 아키텍처 구조를 개선하는 데 도움을 받았습니다.
- **객체 출력 포맷팅:** `toString()` 메서드 오버라이딩과 `String.format()`을 활용하여 콘솔 창에 데이터를 정돈된 형태로 출력하는 문법을 학습했습니다.

## 📂 2. 시스템 메뉴 구조도

본 프로그램은 `Ecommerce.java` (메인 클래스)를 진입점으로 하며, 사용자의 로그인 상태 및 권한(일반회원/관리자)에 따라 접근할 수 있는 메뉴가 동적으로 제한됩니다.

```text
[ e-Commerce 메인 메뉴 ]
 ├── 1. 회원 관리 (비로그인 및 모든 사용자)
 │    ├── 1. 회원가입 (일반회원/관리자 권한 선택 가입)
 │    ├── 2. 로그인 (이메일, 비밀번호 인증)
 │    ├── 3. 로그아웃
 │    ├── 4. 회원 정보 수정 (이름, 전화번호, 이메일 수정)
 │    ├── 5. 비밀번호 변경
 │    └── 0. 뒤로 가기
 │
 ├── 2. 장바구니 메뉴 (로그인한 사용자 전용)
 │    ├── 1. 장바구니 담기 (상품 ID, 수량 입력)
 │    ├── 2. 장바구니 목록 조회 (최신 상품 가격과 조인하여 합계 계산)
 │    ├── 3. 수량 변경
 │    ├── 4. 선택 상품 삭제
 │    ├── 5. 장바구니 비우기 (전체 삭제)
 │    └── 0. 뒤로 가기
 │
 ├── 3. 주문 메뉴 (로그인한 사용자 전용)
 │    ├── 1. 장바구니 선택 상품 주문 (상품 ID 직접 입력하여 주문)
 │    ├── 2. 장바구니 전체 상품 주문 (장바구니 내역을 일괄 주문 후 장바구니 비움)
 │    └── 0. 뒤로 가기
 │
 ├── 4. 카테고리 관리 (관리자 권한 전용)
 │    ├── 1. 카테고리 생성 (대분류/중분류 지원)
 │    ├── 2. 카테고리 수정
 │    ├── 3. 카테고리 삭제
 │    └── 0. 뒤로 가기
 │
 ├── 5. 상품 관리 (관리자 권한 전용)
 │    ├── 1. 상품 등록
 │    ├── 2. 상품 조회
 │    ├── 3. 상품 수정
 │    ├── 4. 상태 변경 (정상/품절 등)
 │    ├── 5. 재고 수량 업데이트
 │    ├── 6. 상품 삭제
 │    └── 0. 뒤로 가기
 │
 └── 0. 프로그램 종료
```

## 🚀 3. 프로그램 실행 및 테스트 설명서

### Step 1. 프로그램 초기화 및 시작
1. IDE(IntelliJ 등)에서 `kr.co.javaex.sec23.Ecommerce` 클래스의 `main` 메서드를 실행합니다.
2. 프로그램이 최초 실행되면 `FileUtil`과 각 `Repository`의 `loadData()`에 의해 JSON 파일(`users.json`, `products.json`, `categories.json` 등)이 없으면 자동으로 생성되며, 과제 명세 조건에 맞는 수십 개의 초기 더미 데이터가 세팅됩니다.

### Step 2. 권한 방어 테스트 (보안성 검증)
1. 프로그램 진입 후 로그인하지 않은 상태에서 **`4` (카테고리 관리)** 또는 **`5` (상품 관리)**를 선택합니다.
2. 다음 메시지가 출력되며 진입이 차단되는지 확인합니다.
   > *"접근 권한이 없습니다. 관리자만 이용 가능합니다."*

### Step 3. 관리자 업무 테스트 (C, R, U, D)
1. **`1` (회원 관리) ➔ `2` (로그인)** 을 선택하여 초기 세팅된 관리자 계정(`admin@test.com` / `admin123!`)으로 접속합니다.
2. **`4` (카테고리 관리)** 및 **`5` (상품 관리)** 메뉴가 정상적으로 열리는지 확인합니다.
3. 상품 등록 시 `상품ID`, `이름`, `가격`, `재고` 등을 입력하여 상품을 등록합니다.
4. **상품 수정 시, 이름이나 설명 입력 단계에서 아무 글자도 치지 않고 `Enter` 키만 누르면 기존 데이터가 그대로 유지되는 것을 확인합니다.**
5. 등록된 상품을 조회/삭제해 보고, 데이터가 정상적으로 `products.json`에 업데이트되는지 확인한 후 로그아웃합니다.

### Step 4. 일반 회원 쇼핑 및 주문 테스트 (데이터 조인 및 동기화 검증)
1. **`1` (회원 관리) ➔ `2` (로그인)** 을 통해 일반 회원(`user1@test.com` / `user123!`)으로 로그인합니다.
2. **`2` (장바구니 메뉴) ➔ `1` (장바구니 담기)** 를 통해 존재하는 상품 ID와 수량을 입력하여 장바구니에 담습니다. *(없는 상품 ID 입력 시 방어 로직 확인)*
3. **`2` (장바구니 목록 조회)** 를 실행하면, 장바구니에 담긴 데이터가 `ProductRepository`의 최신 정보와 **조인(Join)** 되어 상품명, 실시간 가격, 총 결제 합계가 계산되어 출력되는지 확인합니다.
4. **`3` (주문 메뉴) ➔ `2` (장바구니 전체 상품 주문)** 을 선택합니다.
   > **"X건의 장바구니 상품 주문이 완료되었습니다"** 메시지를 확인합니다.
5. 다시 **`2` (장바구니 메뉴)** 로 돌아가 조회를 했을 때, 데이터 연동이 성공적으로 이루어져 다음 메시지가 출력되는지 확인합니다.
   > *"장바구니가 비어있습니다."*

### Step 5. 프로그램 종료
- 메인 메뉴에서 **`0`** 을 입력하여 프로그램을 안전하게 종료합니다. 모든 변경된 데이터는 JSON 파일에 영구적으로 보존됩니다.
